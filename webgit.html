Git Course

Learn Git from Basic to Advanced Concepts

1. Introduction to Git

  • What is Git?
    Git is a distributed version control system for tracking changes in source code.
  • Version Control Systems
    Version control systems manage code changes, with Git enabling collaboration and history tracking.
  • Installing Git
    Install Git on your system to start managing repositories from the command line.

2. Basic Git Commands

  • Initializing a Repository (git init)
    Use `git init` to create a new Git repository in a directory.
  • Staging Files (git add)
    Stage changes with `git add` to prepare files for a commit.
  • Committing Changes (git commit)
    Save staged changes to the repository with `git commit` and a message.

3. Repository Management

  • Checking Status (git status)
    Use `git status` to view the current state of your working directory and staging area.
  • Viewing History (git log)
    Display commit history with `git log` to track changes and authors.
  • Configuring Git (git config)
    Set user details and preferences with `git config` for personalized commits.

4. Working with Branches

  • Creating and Switching Branches (git branch, git checkout)
    Create branches with `git branch` and switch using `git checkout` for isolated work.
  • Merging Branches (git merge)
    Combine branch changes into another branch using `git merge`.
  • Deleting Branches
    Remove unused branches with `git branch -d` to keep the repository clean.

5. Remote Repositories

  • Connecting to Remote Repositories (git remote)
    Link a local repository to a remote one using `git remote add`.
  • Pushing Changes (git push)
    Upload local commits to a remote repository with `git push`.
  • Pulling Changes (git pull)
    Fetch and merge remote changes into your local repository with `git pull`.

6. Undoing Changes

  • Discarding Changes (git restore)
    Discard uncommitted changes in files using `git restore`.
  • Reverting Commits (git revert)
    Create a new commit to undo a previous commit with `git revert`.
  • Resetting Commits (git reset)
    Undo commits with `git reset` to move the branch pointer backward.

7. Advanced Branching

  • Rebasing (git rebase)
    Reapply commits on a new base with `git rebase` for a cleaner history.
  • Cherry-Picking Commits
    Apply specific commits from one branch to another using `git cherry-pick`.
  • Interactive Rebasing
    Modify commit history with `git rebase -i` to squash, edit, or reorder commits.

8. Collaboration Workflows

  • Forking and Pull Requests
    Fork repositories and submit pull requests to contribute to projects.
  • Code Reviews with Git
    Review code changes in pull requests to ensure quality and collaboration.
  • Managing Conflicts
    Resolve merge conflicts manually to integrate conflicting changes.

9. Stashing and Cleaning

  • Stashing Changes (git stash)
    Temporarily save uncommitted changes with `git stash` for later use.
  • Applying and Dropping Stashes
    Reapply or remove stashed changes using `git stash apply` or `git stash drop`.
  • Cleaning Untracked Files (git clean)
    Remove untracked files from the working directory with `git clean`.

10. Git Hosting Platforms

  • Working with GitHub
    Use GitHub to host and collaborate on Git repositories remotely.
  • Using GitLab
    Manage Git repositories on GitLab with built-in CI/CD and project tools.
  • Bitbucket Basics
    Host Git repositories on Bitbucket with features for team collaboration.

11. Advanced Git Commands

  • Bisecting for Debugging (git bisect)
    Use `git bisect` to find the commit introducing a bug via binary search.
  • Submodules (git submodule)
    Manage external repositories within a project using `git submodule`.
  • Reflog for Recovery (git reflog)
    Recover lost commits or branches with `git reflog` to view reference logs.

12. Git Hooks

  • Introduction to Git Hooks
    Git hooks are scripts that run automatically on specific Git events.
  • Client-Side Hooks (pre-commit, pre-push)
    Use client-side hooks like `pre-commit` to enforce checks before actions.
  • Server-Side Hooks
    Server-side hooks control repository actions, like enforcing commit policies.

13. Git Workflows

  • Gitflow Workflow
    Gitflow uses dedicated branches for features, releases, and hotfixes.
  • Feature Branch Workflow
    Develop features in isolated branches, merging via pull requests.
  • Trunk-Based Development
    Commit directly to the main branch for rapid, continuous integration.

14. Automation and CI/CD

  • Integrating Git with CI/CD Pipelines
    Connect Git to CI/CD tools for automated testing and deployment.
  • GitHub Actions Basics
    Automate workflows with GitHub Actions for testing and deployment.
  • GitLab CI/CD
    Use GitLab CI/CD to define pipelines for building and deploying code.

15. Security and Best Practices

  • Signing Commits (GPG)
    Sign commits with GPG to verify their authenticity and integrity.
  • Managing Secrets in Git
    Avoid committing sensitive data by using `.gitignore` and secret scanning.
  • Git Best Practices
    Write clear commit messages and maintain a clean repository history.
